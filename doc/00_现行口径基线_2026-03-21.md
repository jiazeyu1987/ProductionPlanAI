# 现行口径基线（2026-03-21）

## 1. 适用范围
- 适用目录：`doc/plan`、`doc/develop`、`doc/MVP`、`doc/offical_resource`。
- 生效日期：2026-03-21。
- 目的：消除跨文档歧义，提供唯一实施口径。

## 2. 文档优先级（冲突处理）
1. 本文档（`00_现行口径基线_2026-03-21.md`）
2. `doc/develop` 现行开发文档（`01`~`20`）
3. `doc/plan` 现行规划文档（`01`、`03`~`09`、`11`、`12_业务部门外部数据提供规范_P0`、`13_K3_MES字段映射签字版_P0`、`ERP取数清单与测试验证方案`、`排产系统需求分析_P0`）
4. `doc/MVP` 执行文档（用于MVP实施落地；冲突时以前三层为准）
5. 历史/模板/备用/转写文档（仅参考，不作为实施口径）

## 3. 范围与对象
1. 排产主体：`production_order`（生产订单）。
2. 生产用料清单：仅追溯展示，不作为排产驱动主体。
3. 一个销售订单可对应多个订单行、多个产品、多个生产订单。

## 4. 资源与产能口径（最终）
1. 硬约束：`Man + Machine + Material` 同时满足才可开工。
2. 班次口径：`1班次=12小时`。
3. 开班规则：常规 `1天1班`，紧张时 `1天2班（日班+夜班）`。
4. 日可排产能：`日可排产能 = capacity_per_shift * 当日开班数（1或2）`。
5. 同时段占用规则：
   - 同一设备同一时段仅允许一个工艺任务。
   - 同一员工同一时段仅允许一个工艺任务。

## 5. 业务口径（最终）
1. 可排产准入：已审核 + 评审通过 + 未冻结 + 未关闭 + 已投放形成生产订单。
2. 交期字段优先级：`promised_due_date > expected_due_date > requested_ship_date`。
3. 锁单策略：锁单不参与自动重排，仅允许人工调整。
4. 重排范围：默认仅重排未完成工序。
5. 冻结窗：未来24小时任务默认不参与自动重排。

## 6. 规则参数（最终）
1. 优先级权重：
   - `W1 交期紧迫度 = 0.40`
   - `W2 加急标识 = 0.25`
   - `W3 客户优先级 = 0.15`
   - `W4 下单先后 = 0.10`
   - `W5 人工权重 = 0.10`
2. 同分决策：加急 > 承诺交期 > 下单时间 > 订单号字典序。
3. 重排触发阈值：
   - 工序延期预测 > 4小时
   - 订单完成率偏差 > 10%
   - 关键设备异常停机 > 30分钟
4. 重排分级：
   - 轻微偏差：局部重排
   - 重大异常：全局重排

## 7. 日历与SLA口径（最终）
1. 日历规则：默认周日不开班；法定节假日与调休由 `shift_calendar` 维护并覆盖默认规则。
2. SLA起止口径：
   - 交期评审：市场发起评审时间 -> 评审结论时间
   - MRP时效：MRP触发时间 -> 计划订单可用时间
   - 外围时效：车间完工交接时间 -> 外围包装灭菌完成时间
   - 交付时效：承诺交期起算 -> 实际发货完成时间

## 8. 接口口径（最终）
1. 主命名统一使用复数；调用时必须带契约前缀（外部 `/v1`，内部 `/internal/v1`）：
   - `GET /v1/erp/schedule-controls`
   - `GET /v1/erp/sales-order-lines`
   - `GET /v1/erp/production-orders`
   - `GET /v1/erp/sales-order-headers-raw`
   - `GET /v1/erp/sales-order-lines-raw`
   - `GET /v1/erp/production-order-headers-raw`
   - `GET /v1/erp/production-order-lines-raw`
   - `GET /v1/reports/workshop-weekly-plan`
   - `GET /v1/reports/workshop-monthly-plan`
   - `GET /v1/reports/workshop-weekly-plan/export`
   - `GET /v1/reports/workshop-monthly-plan/export`
   - `POST /v1/erp/schedule-results`
   - `POST /v1/erp/schedule-status`
   - `GET /v1/mes/equipments`
   - `GET /v1/mes/reportings`
   - `GET /internal/v1/internal/order-pool`
   - `GET /internal/v1/internal/schedule-versions`
   - `GET /internal/v1/internal/simulation/state`
2. 历史单数路径仅允许兼容，不作为主规范。
3. ERP订单数据模型采用“双层结构”：
   - 主业务事实层：`sales-order-lines`、`production-orders`，仅保留稳定业务字段与 `erp_*` 引用键。
   - 原始明细层：`*-headers-raw`、`*-lines-raw`，完整保留 `erp_header_json` / `erp_line_json`。

## 9. 回写权威口径（最终）
1. ERP回写主对象：`production_order`。
2. 销售订单行/订单头状态由ERP侧按生产订单汇总，不作为排产系统主写入对象。

## 10. 技术栈与部署（最终）
1. MVP默认技术栈：`Java 21 + Spring Boot + PostgreSQL + React`（`Redis`可选）；本地默认运行 profile 为 `local`（H2）。
2. 后端默认运行 profile：`local`（H2 内存库）；需要 PostgreSQL 联调时切换到 PG 模式运行。
3. 当前默认启动形态：脚本启动本地进程栈（`run_stack.ps1` / `code/scripts/run_stack.ps1`），非强制 Docker 编排。
4. 当前默认端口：前端 `5932`，后端 `5931`。
5. 前端默认调用：`http://localhost:5931`；后端 CORS 默认放行：`http://localhost:5932`。
6. P1扩展技术栈与部署：`RabbitMQ + OR-Tools Worker + 独立 sync/worker 进程`。
7. 时区统一：`Asia/Shanghai`。

## 11. 历史文档解释规则
1. 文档名称包含“备用”“确认稿”“模板”“问题清单”“转写”的，默认视为历史或过程记录。
2. 历史文档出现与本基线冲突的表述时，一律以本基线为准。
