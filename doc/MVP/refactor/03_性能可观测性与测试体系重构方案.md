# 03 性能可观测性与测试体系重构方案（v3，状态更新）

> 更新日期：2026-03-24  
> 对齐代码：`MvpStoreService.generateSchedule`、`MvpStoreService.advanceManualOneDay`、`SchedulerBenchmarkSmokeTest`、CI workflows

## 1. 结论
性能可观测性与测试体系已完成第一阶段落地，当前重点是阈值校准与持续观测。

## 2. 已落地能力
1. 排产耗时打点已落地：
   - `schedule_generate_duration_ms`
   - `schedule_generate_phase_duration_ms`
2. 版本算法摘要已回传观测指标：
   - `schedule_completion_rate`
   - `unscheduled_reason_distribution`
   - `locked_or_frozen_impact_count`
   - `publish_rollback_count`
3. 基准测试已落地：
   - `SchedulerBenchmarkSmokeTest`
   - 支持规模参数：100 / 1000 / 5000
   - 支持阈值参数：`-Dmvp.benchmark.thresholds=...`
4. CI 基准门禁已落地：
   - `.github/workflows/backend-benchmark-gate.yml`
   - PR/Push 轻量基准 + 夜间计划基准。

## 3. 现有质量门禁
1. 工件与契约门禁：
   - `.github/workflows/p0-quality-gate.yml`
   - `scripts/check_artifacts.py`
2. 性能门禁：
   - `.github/workflows/backend-benchmark-gate.yml`
   - `SchedulerBenchmarkSmokeTest`

## 4. 当前不足（后续迭代项）
1. 指标尚未接入外部时序系统（当前以接口返回与日志为主）。
2. 前端性能监控（首屏/交互）尚未形成统一上报闭环。
3. 基准阈值需结合真实环境继续校准（当前阈值以 CI 基线为主）。

## 5. 建议下一步
1. 将排产关键指标接入统一监控看板（按版本、按日期聚合）。
2. 增加“未排原因分布波动”告警阈值。
3. 对 `manual/advance-day` 做更细粒度阶段指标对外暴露。
