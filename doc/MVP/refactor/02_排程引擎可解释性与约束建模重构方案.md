# 02 排程引擎可解释性与约束建模重构方案（v3，状态更新）

> 更新日期：2026-03-24  
> 对齐代码：`SchedulerEngine`、`MvpStoreService.listScheduleTasks`、`/internal/.../algorithm`

## 1. 结论
“可解释性基础能力”已在当前代码落地，文档从“问题清单”更新为“现状+增量优化”。

## 2. 已落地能力
1. 未排产已支持结构化原因码，不再只给单一粗粒度结果。
2. 任务明细接口已输出：
   - `dependency_status`
   - `task_status`
   - `last_block_reason`
   - `unscheduled_reason_code`
3. 版本算法说明接口已支持：
   - `summary`
   - `priority_preview`
   - `process_summary`
   - `unscheduled_reason_summary`
   - `unscheduled_samples`
4. 前端已接入算法解释展示（版本中心与调度台）。

## 3. 当前任务状态模型（代码口径）
1. `READY`
2. `PARTIALLY_ALLOCATED`
3. `UNSCHEDULED`
4. `PRESERVED_LOCKED`
5. `SKIPPED_FROZEN`

## 4. 当前依赖状态模型（代码口径）
1. `READY`
2. `WAIT_PREDECESSOR`
3. `BLOCKED_BY_PREDECESSOR`

## 5. 仍可优化项
1. 前端原因文本映射统一：`CAPACITY_UNKNOWN` 与历史 `CAPACITY_LIMIT` 的文案进一步收敛。
2. 解释字段的多语言治理（当前以中文主字段为主，英文仍偏技术语义）。
3. 复杂并行工序下的更细粒度 evidence 输出（当前已具备基础 evidence，但可继续增强）。

## 6. 验收建议
1. 版本中心“算法说明”可稳定展示原因分布与样例。
2. 调度台“任务明细”中可直观看到依赖状态与最后阻塞原因。
3. 回归测试中不可再次退化为“全部未排均显示同一原因”。
