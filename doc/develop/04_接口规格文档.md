# 接口规格文档（开发版）

## 1. 接口清单

路径约定：
- 本文中的接口路径为 OpenAPI 相对路径。
- 实际联调调用时必须补全契约前缀：外部接口使用 `/v1` + 相对路径（例如 `GET /v1/erp/sales-order-lines`）。
- 本文 1.5“内部执行事件”同样属于外部契约路径，完整调用示例：`POST /v1/internal/replan-jobs`。

### 1.1 ERP入站
- `GET /erp/sales-order-lines`
- `GET /erp/sales-order-headers-raw`
- `GET /erp/sales-order-lines-raw`
- `GET /erp/plan-orders`
- `GET /erp/production-orders`
- `GET /erp/production-order-headers-raw`
- `GET /erp/production-order-lines-raw`
- `GET /erp/schedule-controls`
- `GET /erp/mrp-links`
- `GET /erp/delivery-progress`
- `GET /erp/material-availability`（分阶段规则：P0由`schedule-controls`返回齐套状态；P1启用该明细接口）

### 1.2 MES入站
- `GET /mes/equipments`
- `GET /mes/process-routes`
- `GET /mes/reportings`
- `GET /mes/equipment-process-capabilities`
- `GET /mes/employee-skills`
- `GET /mes/shift-calendar`

### 1.3 ERP回写
- `POST /erp/schedule-results`
- `POST /erp/schedule-status`

### 1.4 报表接口（金标准输出）
- `GET /reports/workshop-weekly-plan`
- `GET /reports/workshop-monthly-plan`
- `GET /reports/workshop-weekly-plan/export`（xlsx）
- `GET /reports/workshop-monthly-plan/export`（xls）

### 1.5 内部执行事件（系统内）
- `POST /internal/wip-lots`（工艺完工后生成WIP批次）
- `POST /internal/replan-jobs`（触发重排任务）

## 2. 通用协议
- 认证：Token/Bearer
- 数据格式：JSON
- 时间格式：ISO8601
- 幂等键：`request_id` + 业务主键
- 时区：统一 `Asia/Shanghai`

说明：
- 本文统一使用复数风格路径（如 `schedule-controls`、`schedule-results`）。
- 若历史接口存在单数别名（如 `schedule-control`），可保留兼容层但不作为主规范。

## 3. 分页与增量
- 分页参数：`page`、`page_size`
- 增量参数：`updated_after`
- 排序：`last_update_time asc`
- 同步模式：
  - 触发同步：上游事件触发拉取/推送
  - 夜间补偿：每日固定窗口补偿同步与对账
- 删除/关闭同步语义：
  - 物理删除：禁止作为常规同步方式；如上游发生物理删除，必须发送删除事件
  - 逻辑关闭：通过 `order_status`/`close_flag` 同步
  - 冻结：通过 `frozen_flag` 同步

## 4. 错误码约定
- `200` 成功
- `400` 参数错误
- `401/403` 认证授权失败
- `404` 资源不存在
- `409` 并发冲突
- `429` 频率限制
- `500` 服务异常

## 5. 重试与补偿
1. 5xx/429 自动重试（指数退避，最多3次）。  
2. 超限进入失败队列并告警。  
3. 支持按 `request_id` 重放。
