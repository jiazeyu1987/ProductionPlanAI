# 上线回滚与运行手册（开发版）

## 0. 当前系统口径（2026-03-23）
1. 一键启动：仓库根目录执行 `.\run_stack.ps1`（或 `.\run_stack.bat`）。
2. 默认地址：前端 `http://localhost:5932`，后端 `http://localhost:5931`。
3. 健康检查：`GET http://localhost:5931/api/health`。
4. 运行日志目录：`code/logs/`（`backend-local.*.log`、`frontend-dev.*.log`）。

## 1. 上线前清单
1. 数据同步联调通过（ERP/MES）。  
2. 核心功能测试通过。  
3. 数据库迁移脚本验证通过。  
4. 回滚方案已演练。  
5. 脚本启动栈健康检查通过（frontend `5932` / backend `5931`）。  
6. 班次日历初始化完成（`1班次=12小时`，日班/夜班配置可用）。  
7. 设备-工艺能力矩阵与员工技能矩阵已完成首轮导入。  
8. 物料可用状态同步链路可用（至少支持齐套/缺料标识）。  

## 2. 上线步骤
1. 备份数据库与配置。  
2. 部署应用服务（api内含排产计算与任务调度）。  
3. 执行DB迁移。  
4. 启动服务并通过健康检查（`/api/health`）。  
5. 初始化同步与小流量验证。  
6. 发布首个计划版本并验证回写。  
7. 验证触发同步链路与夜间补偿任务（均由api内置任务执行器承载）。  
8. 启动正式运行与告警通道（钉钉/邮件/系统内消息）。  
9. 执行首日班次演练（1班）与扩容演练（2班）并确认产能口径一致。  

## 3. 回滚步骤
1. 停止发布入口与自动任务。  
2. 切换只读保护数据。  
3. 回退应用版本。  
4. 回退数据库快照。  
5. 恢复旧流程并公告影响。  

## 4. 日常运行
- 每日检查：同步延迟、触发同步成功率、发布成功率、回写失败率、预警数量。  
- 每日检查：当日开班数（1/2班）与计划产能计算是否一致。  
- 每日检查：关键工艺人力覆盖率、关键设备可用率、缺料任务占比。  
- 每周检查：重排次数、延期订单趋势、人工干预占比。  

## 5. 系统侧运行阈值（技术）
- 回写成功率目标：>= 99%。
- 单次排产计算目标：P95 <= 5 分钟。
- 同步策略：触发同步为主，夜间补偿用于漏数修复与对账。
- 超阈值处理：自动告警并转入补偿队列/人工处理。  

## 6. 常见问题处理
### 6.1 同步失败
- 检查鉴权、网络、增量游标。

### 6.2 发布失败
- 检查版本状态、任务锁、回写响应。

### 6.3 重排异常
- 检查规则配置、锁单状态、工艺依赖数据。

### 6.4 班次产能异常
- 检查班次日历是否配置正确（12小时、日班/夜班开班标记）。
- 检查 `capacity_per_shift` 与当日开班数是否匹配。

### 6.5 资源不可执行
- 检查设备-工艺能力矩阵、员工技能矩阵、物料齐套状态是否缺失或过期。

## 7. 审计与追踪
- 所有关键操作必须记录审计日志。  
- 问题排查以 `request_id` 和订单号为主线追踪。
