# 排产引擎详细设计文档

## 1. 输入与输出

### 1.1 输入
- 可排产生产订单集合（订单号、数量、交期、加急、冻结/锁定状态）
- 计划订单投放关联（用于来源追溯，不参与排产打分）
- 产品工艺路线（工序顺序、串并行关系）
- 设备/工序产能（班次能力、日历可用性）
- 设备-工艺能力矩阵、员工技能矩阵、物料可用状态
- 人工调度指令（插单、调序、锁单）

### 1.2 输出
- 工序级计划任务（开始时间、结束时间、计划量）
- 订单级预计完工时间
- 瓶颈工序与超负荷信息
- 资源约束不足清单（人力缺口、设备冲突、缺料）
- WIP批次生成规则（按工序完工形成）
- 版本结果（草稿/发布）

## 2. 计算流程
1. 准入过滤：剔除不可排产、冻结订单。  
2. 排产主体过滤：仅保留生产订单，剔除非生产类订单对象。  
3. 人机料可执行性校验：过滤不可开工任务并标注原因。  
4. 优先级计算：生成订单排序分值。  
5. 粗排：按产品线/关键工序分配能力。  
6. 细排：按工序依赖、班次日历分配时段。  
7. 冲突检测：产能冲突、依赖冲突、资源冲突。  
8. 生成版本：输出任务清单、影响分析与风险摘要。  

## 3. 优先级评分模型（示例）
`Score = W1*交期紧迫度 + W2*加急标识 + W3*客户优先级 + W4*下单先后 + W5*人工权重`

- 交期越近分越高
- 加急订单加固定分
- 人工干预可设置附加权重

同分决策（Tie-break）：
1. 先比较加急标识  
2. 再比较承诺交期  
3. 再比较下单时间  
4. 再比较订单号字典序（保证结果稳定可复现）

## 4. 产能与班次口径
1. 班次口径固定：`1班次=12小时`。  
2. 每日开班数：`1`（常规）或`2`（日班+夜班）。  
3. 日可排产能：`日可排产能 = capacity_per_shift * 当日开班数`。  
4. 工艺可执行组数：
   - `机器可开组数 = floor(可用设备数 / required_equipment_count)`
   - `人力可开组数 = floor(可用熟练工数 / required_manpower_per_group)`
   - `可执行组数 = min(机器可开组数, 人力可开组数, 物料可开组数)`  
5. 工艺班次产能：`工艺班次产能 = 可执行组数 * 单组标准班次产能`。  

## 5. 工序调度规则
### 5.1 串行
- 后置工序开始时间 >= 前置工序完成时间

### 5.2 并行
- 同并行组工序可同窗口并发安排
- 并发上限由设备可用数决定

### 5.3 日历约束
- 非工作时段不可排
- 停机窗口不可排
- 支持跨天跨班次自动续排

## 6. 重排机制
- 触发条件：
  1. 报工偏差超过阈值
  2. 关键设备异常
  3. 新插单进入
- 重排范围：
  - 默认仅重排“未完成工序”
  - 锁单订单不参与自动重排
- 分级策略：
   - 轻微偏差：局部重排
   - 重大异常：全局重排
- 冻结窗规则：
   - 未来24小时任务默认冻结，自动重排不改，仅允许审批后人工调整

优先级覆盖顺序：
`冻结/锁单规则 > 人工指令 > 系统评分规则 > 默认排序`

## 7. 影响分析输出
- 受影响订单数
- 新增延期订单数
- 最大延期时长
- 瓶颈工序变化
- 资源缺口变化（人力/设备/物料）

## 8. 性能与稳定性要求
- 单次排产计算 P95 <= 5 分钟。
- 失败可重算，结果可复现（同输入同版本）。
- 支持并发请求下的任务锁与排队机制。
