# 测试自动化与验收工件规范（P0，现行）
> 文档属性：现行测试规范。  
> 生效日期：2026-03-22。

## 1. 目的
1. 将测试方案从“清单级”落到“可执行自动化+可签署验收”。
2. 建立从需求到测试到验收证据的闭环追踪。

## 2. 自动化测试分层与门槛
1. 单元测试覆盖率：核心排产域 `>=80%`。
2. 集成测试：ERP/MES入站、回写、幂等、重试、补偿全部自动化。
3. E2E：`接单->排产->发布->报工->重排` 主链路自动化。
4. 回归门槛：阻断/高优先缺陷必须为0。

## 3. 用例到自动化映射（最小集）
1. `TC-001` 正常排产发布 -> `AT-E2E-001`。
2. `TC-002` 插单影响分析 -> `AT-E2E-002`。
3. `TC-003` 锁单不自动重排 -> `AT-E2E-003`。
4. `TC-006` 报工更新进度 -> `AT-INT-006`。
5. `TC-007` 延期触发重排 -> `AT-E2E-007`。
6. `TC-015` 班次产能口径 -> `AT-UNIT-015 + AT-E2E-015`。
7. `TC-016` 人机料约束 -> `AT-E2E-016`。
8. `TC-020` WIP唯一批次 -> `AT-INT-020`。
9. `TC-021` 局部/全局重排 -> `AT-E2E-021`。
10. `TC-022` 冻结窗校验 -> `AT-E2E-022`。

## 4. 测试数据资产
1. 标准数据包目录：`testdata/p0_baseline_pack/`。
2. 异常数据包目录：`testdata/p0_negative_pack/`。
3. 在制迁移数据包目录：`testdata/p0_cutover_inprogress_pack/`。
4. 数据包必须版本化并记录生成脚本与种子。

## 5. CI/CD 质量闸门
1. 每次合并必须通过：`unit + integration + contract test`。
2. 每日构建必须执行：`E2E smoke`。
3. 发布前必须执行：`full regression + performance smoke`。
4. 任一关键闸门失败禁止进入发布流程。

## 6. 验收工件清单（必须留档）
1. 测试报告：`test_report_YYYYMMDD.md`。
2. 覆盖率报告：`coverage_YYYYMMDD.html`。
3. 接口契约校验报告：`contract_check_YYYYMMDD.json`。
4. 性能摘要：`perf_summary_YYYYMMDD.md`。
5. 缺陷清单与关闭证明。
6. UAT签署单（业务+计划+信息化三方签字）。

## 7. UAT签署标准
1. 核心流程通过率 `100%`。
2. 阻断/高优先缺陷 `0`。
3. 回写成功率与同步成功率达到门槛。
4. 关键业务部门书面签署通过。

## 8. 回归基线管理
1. 每次版本发布后固化“回归基线版本号”。
2. 参数版本变更必须触发回归。
3. 接口契约变更必须触发契约回归。
