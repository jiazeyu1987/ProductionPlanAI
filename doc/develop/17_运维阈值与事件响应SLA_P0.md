# 运维阈值与事件响应SLA（P0，现行）
> 文档属性：现行运行规范。  
> 生效日期：2026-03-22。

## 1. 目的
1. 固化监控阈值、告警分级、响应时限。
2. 明确值班责任与升级路径，避免“发现问题但无人处理”。

## 2. 指标阈值

### 2.1 同步链路
1. ERP/MES同步延迟：
   - 告警：`>10分钟`
   - 严重：`>30分钟`
2. 触发同步成功率（5分钟窗口）：
   - 告警：`<99%`
   - 严重：`<97%`
3. 夜间补偿成功率：
   - 告警：`<99.5%`
   - 严重：`<98%`

### 2.2 排产与发布
1. 单次排产P95：
   - 告警：`>5分钟`
   - 严重：`>8分钟`
2. 发布成功率（日）：
   - 告警：`<99.5%`
   - 严重：`<98%`
3. ERP回写成功率（日）：
   - 告警：`<99%`
   - 严重：`<97%`

### 2.3 业务稳定性
1. 重排失败率（日）：
   - 告警：`>3%`
   - 严重：`>8%`
2. 不可执行任务占比（日）：
   - 告警：`>10%`
   - 严重：`>20%`
3. 关键工艺人力覆盖率：
   - 告警：`<95%`
   - 严重：`<90%`

## 3. 告警分级与响应SLA
1. `P1`（生产阻断）：5分钟响应，30分钟给出处置方案。
2. `P2`（核心功能退化）：15分钟响应，2小时内恢复。
3. `P3`（一般异常）：30分钟响应，8小时内恢复。
4. `P4`（提示类）：工作日内处理。

## 4. 处置RACI
1. 值班工程师（R）：告警接单、初步定位、执行处置。
2. 技术负责人（A）：升级决策、回滚批准、跨团队协调。
3. 业务计划负责人（C）：确认业务影响与临时策略。
4. 运维/信息化（C）：基础设施、网络、数据库处置。
5. 管理层（I）：接收P1/P2事件通报。

## 5. 标准处置流程
1. 告警触发 -> 5分钟内确认。
2. 判断级别 -> 启动对应应急群。
3. 执行预案（重试/补偿/回滚/限流）。
4. 恢复后 24 小时内输出复盘报告。

## 6. 预案清单（必须预置）
1. 同步中断预案。
2. 发布失败预案。
3. 回写失败预案。
4. 重排异常预案。
5. 数据一致性异常预案。

## 7. 演练要求
1. 每月至少1次 P1/P2 演练。
2. 每季度至少1次全链路切换回滚演练。
3. 演练必须产出记录、问题清单、改进闭环。

## 8. 事件复盘模板（最小字段）
1. 事件编号
2. 发生时间/恢复时间
3. 影响范围（订单数、时长）
4. 根因
5. 临时修复
6. 永久修复
7. 责任人与截止日期
