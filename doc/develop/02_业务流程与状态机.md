# 业务流程与状态机（排产系统）

## 1. 业务主流程
1. ERP销售订单与MRP计划订单链路入站。  
2. 计划订单投放后生成生产订单 -> 进入待排池。  
3. 校验可排产条件（准入状态 + 人机料约束 + 班次日历）-> 进入可排清单。  
4. 排产引擎计算（粗排+细排）-> 生成草稿版本。  
5. 计划员调序/插单/锁单 -> 形成发布版本。  
6. 发布后回写 ERP。  
7. MES 报工入站 -> 实绩对比。  
8. 偏差触发重排 -> 生成新候选版本。  
9. 人工确认候选版本 -> 发布生效版本。  

## 2. 订单状态机
- `NEW`：新入站未校验  
- `PENDING`：待排产  
- `SCHEDULED`：已进入某版本  
- `RUNNING`：执行中（有报工）  
- `DONE`：完成  
- `EXCEPTION`：异常  
- `FROZEN`：冻结（禁止自动变更）
- `ARCHIVED`：关闭归档（历史只读）

状态迁移：
- `NEW -> PENDING`（校验通过）
- `PENDING -> SCHEDULED`（版本排入）
- `SCHEDULED -> RUNNING`（开始报工）
- `RUNNING -> DONE`（完成量达到计划）
- 任意状态 -> `EXCEPTION`（异常事件）
- `PENDING/SCHEDULED/RUNNING -> FROZEN`（人工冻结）
- `FROZEN -> PENDING`（人工解冻且未排入有效版本）
- `FROZEN -> SCHEDULED`（人工解冻且仍在生效版本中）
- `EXCEPTION -> PENDING`（异常处理完成，重新待排）
- `EXCEPTION -> RUNNING`（异常不影响继续执行）
- `DONE -> ARCHIVED`（订单关闭归档）

## 3. 版本状态机
- `DRAFT`：草稿  
- `READY`：待发布  
- `PUBLISHED`：已发布  
- `ARCHIVED`：历史版本  
- `ROLLED_BACK`：回滚版本

状态迁移：
- `DRAFT -> READY`（完成校验）
- `READY -> PUBLISHED`（发布）
- `PUBLISHED -> ARCHIVED`（新版本发布）
- `PUBLISHED -> ROLLED_BACK`（回滚）
- `ROLLED_BACK -> PUBLISHED`（恢复到回滚目标版本）

## 4. 重排触发状态
- `NONE`：无重排  
- `TRIGGERED`：触发重排条件  
- `RECALCULATED`：已完成重算  
- `CONFIRMED`：人工确认采用  
- `REJECTED`：人工拒绝候选版本

## 5. 关键规则
1. `FROZEN` 订单不参与自动重排。  
2. 锁单仅允许人工调整。  
3. 重排只影响“未完成工序”。  
4. 发布版本必须唯一生效。  
5. 回滚不删除历史版本，必须保留审计链路。  
6. 解冻与异常恢复必须填写原因并记录审批结果。  
7. 排产计算主体为生产订单，生产用料清单仅用于追溯展示。  
8. 班次口径固定：`1班次=12小时`，当日开班数为`1`或`2`。  
9. 工序可开工前需同时满足人力、设备、物料三类约束。
