# 开发版PRD（排产系统）

## 1. 项目目标
- 建设以订单交付为目标的排产系统，替代 Excel 主流程。
- 打通 ERP（K3）订单侧数据与 MES 执行侧数据。
- 实现“排产-发布-执行反馈-重排”闭环。
- 贯通 K3 的 MRP 链路（销售订单 -> 计划订单投放 -> 生产订单/采购申请）到排产主流程。

## 2. 范围与边界
### 2.1 范围内
- 订单待排池、规则排序、有限产能排产、人工调序、版本发布、报工反馈、自动重排、预警看板。
- 工艺级任务编排（串并行工序）、人机料约束校验与瓶颈识别。
- 班次化产能模型与班次日历管理。

### 2.2 范围外
- 库存可用量精算（安全库存/即时库存扣减优化）。
- 成本最优排产、财务结算相关能力。

## 3. 关键业务约束
1. 只处理“可排产订单”（已审核、评审通过、未冻结、未关闭、已投放形成生产订单）。
2. 排产对象为生产订单；生产用料清单为一对一附属数据，不作为排产驱动对象。
3. 销售订单可包含多个产品（多订单行），排产需下沉到对应生产订单与工艺任务。
4. 排产硬约束为 `Man + Machine + Material`：人力、设备、物料需同时满足才可开工。
5. 班次口径固定：`1班次=12小时`，常规 `1天1班`，紧张时可 `1天2班（日班+夜班）`。
6. 日产能统一公式：`日可排产能 = capacity_per_shift * 当日开班数（1或2）`。
7. 支持工序串行/并行约束、换型损耗、停机窗口与跨班次续排。
8. 支持加急插单、锁单、人工调整留痕；锁单默认不参与自动重排。
9. 排产输出需支持版本化（V1/V2...），同一时刻仅允许一个生效版本。
10. 官方流程时效（SLA）必须内置监控：
   - 交期评审：1个工作日内完成
   - 常规订单交付：45天内完成
   - MRP运算：常规2个工作日、加急1个工作日
   - 外围包装灭菌：收到产品后3天内完成
11. 同步机制采用“触发同步 + 每晚定时补偿同步”。
12. 调序/发布/回滚/解冻权限可配置，关键动作强审计。

## 4. 核心功能
1. 订单待排池与筛选（交期、客户、产品、状态、加急、冻结、锁单）。
2. 排产规则引擎（交期优先、加急优先、客户优先、人工权重）。
3. 自动排产（粗排+细排），输出工序级任务与订单级预计完工时间。
4. 资源约束校验（设备-工艺能力、员工技能、物料齐套）与瓶颈分析。
5. 调度台（拖拽调序、插单、锁单、回滚、影响预览）。
6. 版本管理（发布、对比、回滚）与关键动作审计。
7. MES 报工接入、进度更新与偏差重排（生成候选版本）。
8. 预警（延期、超负荷、异常停机、缺料、技能缺口）。
9. 交付控制（订单交期与发货日期分离、分批发货通知）。
10. 外围节点跟踪（外围包装、灭菌、入库进度可视化）。

## 5. 非功能需求
- 可用性：核心流程稳定可用，支持试点车间连续运行。
- 可追溯：关键操作全留痕，支持按订单/版本/request_id追踪。
- 性能：单次排产计算 `P95 <= 5 分钟`。
- 安全：按角色授权访问，关键操作审计。
- 告警：支持钉钉、邮件、系统内消息三通道。

## 6. 验收标准
1. 系统可独立完成周排产并发布，不依赖 Excel 主流程。
2. 报工后可自动更新计划进度并生成重排候选版本。
3. 班次口径与产能计算一致（12小时/班，1或2班）。
4. 插单/锁单可执行且有完整审计记录。
5. 排产结果可回写 ERP 并可追踪。
6. SLA 监控可查询并告警（评审、MRP、外围、交付）。
