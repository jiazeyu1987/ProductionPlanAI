# 排产精细化补充决议（2026-03-21，现行）
> 文档属性：现行治理文档。  
> 口径来源：`doc/00_现行口径基线_2026-03-21.md`。

## 1. 适用范围
- 适用于排产系统 P0 设计、开发、测试、上线与运维。
- 本文中的“规则/必须/默认”均为现行执行口径。
- 本文中的“迭代路线”用于划分版本边界，不属于未决事项。

## 2. 已确认统一口径

### 2.1 班次与产能
1. 产能按班次计算。
2. `1班次 = 12小时`。
3. 开班规则：常规 `1天1班`，紧张时 `1天2班（日班+夜班）`。
4. 统一计算式：`日可排产能 = capacity_per_shift * 当日开班数`，其中当日开班数取值为 `1` 或 `2`。

### 2.2 排产约束与对象
1. 排产硬约束固定为 `Man + Machine + Material`。
2. 单产品生产过程可拆分为多个工艺；必经工艺全部完成后产品才算完成。
3. 一个销售订单可对应多个产品（多订单行），排产执行对象固定为 `production_order`。
4. 同一设备可具备多个工艺能力；同一员工可具备多个工艺技能。

## 3. 设计执行规则（P0）

### 3.1 工艺级产能计算规则
1. `机器可开组数 = floor(可用设备数 / required_equipment_count)`。
2. `人力可开组数 = floor(可用熟练工数 / required_manpower_per_group)`。
3. `可执行组数 = min(机器可开组数, 人力可开组数, 物料可开组数)`。
4. `工艺班次产能 = 可执行组数 * 单组标准班次产能`。
5. `工艺日可排产能 = 工艺班次产能 * 当日开班数`。

### 3.2 资源模型规则
1. 必须维护 `设备-工艺能力矩阵` 与 `员工-工艺技能矩阵`。
2. 同一时段同一设备仅允许一个工艺任务。
3. 同一时段同一员工仅允许一个工艺任务。
4. 排程必须计入换型/清线时间与停机窗口。

### 3.3 中间在制品（WIP）规则
1. 每道工艺完工后必须生成唯一 `wip_lot_id`。
2. WIP 粒度固定为“批次（lot）”，不使用“单件”粒度。
3. WIP 用于转序、重排未完工识别、质量追溯、返工与报废闭环。

### 3.4 业务对象分解规则
1. `sales_order`（订单头）
2. `sales_order_line`（订单行）
3. `production_order`（排产主体）
4. `process_task`（工艺任务）
5. `wip_lot`（中间在制批次）

### 3.5 重排与冻结窗规则
1. 未来24小时任务默认处于冻结窗，自动重排不改动。
2. 冻结窗内任务仅允许审批后人工调整。
3. 重排分级固定为：轻微偏差局部重排、重大异常全局重排。

## 4. Dashboard 必显指标（P0）
1. 准交率
2. 延期订单数
3. 计划达成率
4. 未来7天延期风险订单数
5. 超负荷工序/产线数
6. 重排次数
7. ERP回写成功率
8. 排产计算 P95

## 5. 迭代路线（版本边界已冻结）
1. P1：换型时间矩阵、订单拆批并行、缺料占位、发布前数据质量闸门。
2. P2：What-if 仿真、规则版本化管理（权重/阈值/班次策略）。
3. 迭代项不影响 P0 现行规则与验收标准。

## 6. 变更控制
1. 本文与其他文档出现冲突时，以 `doc/00_现行口径基线_2026-03-21.md` 为准。
2. 任何口径变更必须先更新基线文档，再同步更新 `doc/plan` 与 `doc/develop`。
