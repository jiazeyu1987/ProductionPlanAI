# 排产引擎参数包与降级策略（P0，现行）
> 文档属性：现行开发规范。  
> 生效日期：2026-03-22。  
> 口径来源：`doc/00_现行口径基线_2026-03-21.md`。

## 1. 目的
1. 固化排产引擎参数，保证“同输入同结果”。
2. 明确参数边界、默认值、生效优先级、发布方式。
3. 明确求解失败时的降级策略，避免生产阻断。

## 2. 参数优先级
`冻结/锁单规则 > 人工指令 > 规则参数版本(rule_param_version) > 系统默认值`

## 3. 参数清单（P0）

### 3.1 评分参数
1. `W1_due_urgency`：默认 `0.40`，范围 `[0,1]`。
2. `W2_urgent_flag`：默认 `0.25`，范围 `[0,1]`。
3. `W3_customer_priority`：默认 `0.15`，范围 `[0,1]`。
4. `W4_order_time`：默认 `0.10`，范围 `[0,1]`。
5. `W5_manual_weight`：默认 `0.10`，范围 `[0,1]`。
6. 约束：`W1+W2+W3+W4+W5 = 1`。

### 3.2 重排参数
1. `delay_threshold_hours`：默认 `4`（触发局部重排）。
2. `progress_gap_threshold`：默认 `0.10`（10%）。
3. `equipment_down_threshold_minutes`：默认 `30`。
4. `freeze_window_hours`：默认 `24`。
5. `replan_scope_mode`：`AUTO`（轻微局部/重大全局）。

### 3.3 资源约束参数
1. `require_man_machine_material_all=true`（三类硬约束全满足）。
2. `allow_material_placeholder=false`（P0默认不可执行占位）。
3. `employee_multi_task_same_slot=false`。
4. `equipment_multi_task_same_slot=false`。
5. `setup_time_mode=PROCESS_STD`（按工艺标准准备时间）。

### 3.4 班次参数
1. `shift_hours=12`（固定）。
2. `max_shifts_per_day=2`。
3. `default_shifts_per_day=1`。

## 4. Tie-break 固化
同分必须按以下顺序比较：
1. 加急标识（高优先）。
2. 承诺交期（早优先）。
3. 下单时间（早优先）。
4. 订单号字典序（小优先）。

## 5. 参数版本发布机制
1. 参数存储在 `rule_param_version`。
2. 版本状态：`DRAFT -> ACTIVE -> ARCHIVED`。
3. 一个时间点仅允许一个 `ACTIVE` 版本。
4. 参数生效必须绑定到 `schedule_version.rule_version_no`。
5. 发布必须记录：发布人、发布时间、变更原因、变更前后值。

## 6. 参数变更管控
1. 变更前必须跑回归用例（至少关键用例1/2/3/7/14/15/16/21/22）。
2. 变更后必须出影响分析：延期订单变化、超负荷变化、重排次数变化。
3. 禁止在生产时段直接改 `ACTIVE` 参数；需通过新版本切换。

## 7. 求解降级策略（必须实现）

### 7.1 触发条件
1. 单次求解超时（> `solver_timeout_seconds`，默认 300 秒）。
2. 求解失败（无可行解）。
3. 关键输入缺失（工艺路线或班次日历缺失）。

### 7.2 降级层级
1. `L1`：缩小范围（仅未完成工序 + 非冻结订单）。
2. `L2`：仅局部重排（受影响订单集合）。
3. `L3`：启发式排程（先交期后资源，放弃全局最优）。
4. `L4`：保留上个已发布版本并生成异常告警，不自动发布新版本。

### 7.3 降级审计
每次降级必须记录：
1. `job_id`
2. 触发原因
3. 降级层级
4. 影响订单数
5. 最终结果（成功/失败）

## 8. 引擎验收标准
1. 同输入同参数版本，结果可复现。
2. 参数非法时禁止发布（如权重和不为1）。
3. 触发降级时系统不中断，且有可追溯日志。
4. 所有发布版本可追溯到参数版本。
