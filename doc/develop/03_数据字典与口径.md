# 数据字典与口径（排产系统）

## 1. 数据来源
- ERP(K3)：订单、交期、状态、MRP关联（销售/计划/生产链路）、交付状态
- MES：设备、工艺路线、工序、报工
- 人工：插单、锁单、优先级调整、异常决策
- 主数据维护：班次日历、技能矩阵、设备-工艺能力矩阵

## 2. 核心实体字段

### 2.1 销售订单行（sales_order_line）
- `sales_order_no` string 必填
- `line_no` string 必填
- `customer_code` string
- `customer_name` string
- `product_code` string 必填
- `product_name` string
- `spec_model` string
- `order_qty` decimal 必填
- `order_date` datetime 必填
- `expected_due_date` datetime 必填
- `promised_due_date` datetime
- `requested_ship_date` datetime
- `urgent_flag` bool(0/1) 必填
- `order_status` string 必填
- `last_update_time` datetime 必填
- `market_note` string（市场备注信息）
- `purchase_due_date` datetime（采购交期）
- `injection_due_date` datetime（注塑交期）
- `workshop_outer_pack_date` datetime/string（车间外围包装日期）

主键：`sales_order_no + line_no`

### 2.2 生产订单（production_order）
- `production_order_no` string 必填
- `source_sales_order_no` string
- `source_line_no` string
- `source_plan_order_no` string（来源计划订单号）
- `material_list_no` string（对应生产用料清单号）
- `product_code` string 必填
- `plan_qty` decimal 必填
- `plan_start_date` datetime
- `plan_finish_date` datetime
- `production_status` string 必填
- `last_update_time` datetime 必填
- `finished_qty_workshop` decimal（完工数量-车间）
- `finished_time_workshop` datetime（完工时间-车间）
- `finished_qty_outer` decimal（完工数量-外围）
- `finished_time_outer` datetime（完工时间-外围）
- `match_order_no` string（匹配单号/关联单号）

主键：`production_order_no`

### 2.3 计划订单（plan_order）
- `plan_order_no` string 必填
- `source_sales_order_no` string
- `source_line_no` string
- `release_type` enum(`PURCHASE`,`OUTSOURCE`,`PRODUCTION`) 必填
- `release_status` string 必填
- `release_time` datetime
- `last_update_time` datetime 必填

主键：`plan_order_no`

### 2.4 工序路线（process_route）
- `route_no` string（路线版本号）必填
- `product_code` string 必填
- `process_code` string 必填
- `process_name` string
- `sequence_no` int 必填
- `dependency_type` enum(`FS`,`SS`) 必填
- `parallel_group` string
- `std_cycle_time` decimal（标准节拍，秒/件）
- `capacity_per_shift` decimal（班次标准产能）
- `required_manpower_per_group` int（单组所需人数）
- `required_equipment_count` int（单组所需设备数）
- `setup_time_minutes` int（换型/准备时间）
- `enabled_flag` bool(0/1)

主键：`route_no + process_code`

### 2.5 设备主数据（equipment）
- `equipment_code` string 必填
- `equipment_name` string
- `line_code` string
- `workshop_code` string
- `status` enum(`RUNNING`,`DOWN`,`MAINTENANCE`,`DISABLED`) 必填
- `capacity_per_shift` decimal
- `last_update_time` datetime 必填

主键：`equipment_code`

### 2.6 设备-工艺能力矩阵（equipment_process_capability）
- `equipment_code` string 必填
- `process_code` string 必填
- `enabled_flag` bool(0/1) 必填
- `capacity_factor` decimal（设备相对产能系数，默认1）
- `last_update_time` datetime 必填

主键：`equipment_code + process_code`

### 2.7 员工技能矩阵（employee_skill）
- `employee_id` string 必填
- `employee_name` string
- `team_code` string
- `process_code` string 必填
- `skill_level` enum(`INDEPENDENT`,`ASSIST`,`NONE`) 必填
- `efficiency_factor` decimal（效率系数，默认1）
- `active_flag` bool(0/1) 必填
- `last_update_time` datetime 必填

主键：`employee_id + process_code`

### 2.8 班次日历（shift_calendar）
- `calendar_date` date 必填
- `shift_code` enum(`DAY`,`NIGHT`) 必填
- `shift_start_time` datetime 必填
- `shift_end_time` datetime 必填
- `open_flag` bool(0/1) 必填
- `workshop_code` string
- `last_update_time` datetime 必填

主键：`calendar_date + shift_code + coalesce(workshop_code,'')`

### 2.9 物料可用（material_availability）
- `material_code` string 必填
- `order_no` string
- `process_code` string
- `available_qty` decimal 必填
- `available_time` datetime
- `ready_flag` bool(0/1) 必填
- `last_update_time` datetime 必填

主键：`material_code + coalesce(order_no,'') + coalesce(process_code,'')`

### 2.10 报工事实（reporting_fact）
- `report_id` string 必填
- `order_no` string 必填
- `order_type` enum(`sales`,`production`) 必填
- `process_code` string 必填
- `report_qty` decimal 必填
- `report_time` datetime 必填
- `shift_code` enum(`DAY`,`NIGHT`)
- `team_code` string
- `operator_code` string
- `exception_code` string
- `created_at` datetime

主键：`report_id`

### 2.11 中间在制批次（wip_lot）
- `wip_lot_id` string 必填
- `order_no` string 必填
- `process_code` string 必填
- `qty` decimal 必填
- `status` enum(`READY`,`IN_TRANSFER`,`HOLD`,`SCRAP`,`CONSUMED`) 必填
- `source_report_id` string
- `create_time` datetime 必填
- `last_update_time` datetime 必填

主键：`wip_lot_id`

### 2.12 调度指令（dispatch_command）
- `command_id` string 必填
- `command_type` enum(`INSERT`,`LOCK`,`UNLOCK`,`PRIORITY`,`FREEZE`,`UNFREEZE`) 必填
- `target_order_no` string 必填
- `effective_time` datetime 必填
- `reason` string 必填
- `created_by` string 必填
- `approved_flag` bool(0/1)
- `created_at` datetime 必填

主键：`command_id`

## 3. 口径规则
1. 时间统一使用 ISO8601（含时区 `Asia/Shanghai`）。
2. 布尔字段统一 `0/1`。
3. 数量字段非负；百分比/系数字段默认 `1` 表示不调整。
4. 订单状态变更以 `last_update_time` 最新值为准。
5. 口头通知必须入 `dispatch_command` 才生效。
6. `workshop_outer_pack_date` 允许文本格式（如“3.26加急”），入库前需标准化。
7. 车间/外围完工数量与报工累计值需可对账。
8. 排产计算以 `production_order` 为主体，不直接基于生产用料清单。
9. 班次口径固定：`1班次=12小时`，每日开班数为 `1` 或 `2`。
10. 日可排产能公式：`日可排产能 = capacity_per_shift * 当日开班数`。
11. 工艺可开工前提：人力、设备、物料三类约束必须同时满足。
12. 销售订单可包含多个订单行（多个产品）；一个订单行可拆分为多个生产订单。
13. `wip_lot_id` 必须全局唯一，用于追踪工艺间转序、冻结、报废与消耗。
14. 工序依赖枚举统一使用 `FS/SS`；若上游仍输出 `SERIAL/PARALLEL`，必须在接入层映射后落库。
