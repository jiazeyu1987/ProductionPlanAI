# 执行顺序与验收清单

## 并行启动
1. A_backend_erp_loader_prompt.md
2. B_backend_erp_data_manager_prompt.md
3. C_frontend_lite_scheduler_prompt.md
4. D_frontend_schedule_calendar_prompt.md
5. E_frontend_masterdata_prompt.md

## 每个子Agent必须输出
1. 变更文件列表
2. 主文件行数 before/after
3. 测试命令和结果（必须贴实际执行情况）
4. 风险与未完成项

## 总负责人集成验收
1. 逐个集成，先后端再前端。
2. 每集成一个子Agent结果，立即运行对应门禁。
3. 全部集成后跑全量门禁：
   - 后端：`cd code/backend && mvn test`
   - 前端：`cd code/frontend && npm test -- --run && npm run build`
4. 记录到 `doc/refactor/test-records/`。

## 阻塞处理规则
1. 某子Agent门禁不通过：先冻结该模块，其他模块继续。
2. 若出现跨模块冲突：以“接口契约不变”和“可回滚”优先。
3. 任何功能性变更必须拆成单独后续任务，不并入本轮拆分。
