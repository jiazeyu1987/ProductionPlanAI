# 璞慧生产业务流程梳理（独立文档）

更新时间：2026-03-28  
范围：`code/frontend/src/pages/LiteSchedulerPage.jsx` + `code/frontend/src/utils/liteSchedulerEngine.js`

## 1. 模块定位

璞慧生产（当前前端入口：`/schedule/lite`）是独立于主后端排产链路的“轻量本地排产”模块，核心特征：

1. 纯前端本地运行。
2. 使用浏览器 `localStorage` 保存场景与快照。
3. 不依赖后端接口进行排产计算与结果持久化。

## 2. 参与角色与目标

1. 计划员：快速录入订单、产线能力、锁定片段，生成排程草案。
2. 现场管理：按日推进模拟、在按天模式下手动报结束、导出排产表。

## 3. 业务主流程

### 3.1 场景初始化与加载

1. 页面加载时读取本地场景（`liteScheduler.scenario.v1`）。
2. 若本地无场景或异常，自动生成默认场景。
3. 场景统一经过 `normalizeLiteScenario` 归一化后进入运行态。

### 3.2 基础排程规则设置

1. 设置排产起始日（`horizonStart`）。
2. 设置规划期长度（`horizonDays`）。
3. 切换排程模式：
   1. `QTY_CAPACITY`：按工作量/产能排。
   2. `DURATION_MANUAL_FINISH`：按天数排，支持手动报结束。
4. 设置日历规则：
   1. 是否跳过法定节假日。
   2. 周末模式（无休/单休/双休）。
   3. 指定日期手工设为“排产/休息”。

### 3.3 产线管理流程

1. 新增产线（名称、基础日产能）。
2. 维护产线基础日产能。
3. 按日期维护产线产能覆盖值（capacity override）。
4. 删除产线前校验：
   1. 该产线无锁定片段。
   2. 该产线无订单工作量/计划天数绑定。

### 3.4 订单管理流程

1. 订单录入（订单号、产品、规格、批号、交期、释放日）。
2. 两种模式下的订单录入差异：
   1. 按量模式：录入各产线工作量。
   2. 按天模式：录入各产线计划天数（可同时录入计划数量用于展示）。
3. 编辑订单时会重算可排工作量并约束已完工量上限。
4. 删除订单前需校验不存在锁定片段。

### 3.5 锁定片段流程（按量模式生效）

1. 为“订单-产线-日期区间”增加锁定工作量。
2. 排程时先消费锁定片段，再做自动分配。
3. 锁定片段可删除。
4. 在按天模式下，锁定片段被忽略（引擎会给 warning）。

### 3.6 自动排产与看板展示

1. 场景变更后自动重算排程（`useMemo + buildLiteSchedule`）。
2. 输出内容：
   1. 按日按线分配明细（calendar/grid）。
   2. 产线负荷汇总。
   3. 订单完成/剩余/延期情况。
   4. 警告信息（非法引用、模式不匹配等）。

### 3.7 推进一天流程

1. 点击“推进一天”调用 `advanceLiteScenarioOneDay`。
2. 系统将当日排程转为“已完成工作量”，并把起始日滚动到下一工作日。
3. 同步更新：
   1. 订单已完成量。
   2. 锁定片段剩余量与起始日。
   3. 模拟日志（最近 30 条）。

### 3.8 按天模式手动报结束流程

1. 在按天模式可对“产线-订单段”设置手动结束日期。
2. 手动结束后，后续订单会按新结束点前移/后延。
3. 支持清除手动结束设置并恢复自动推导。

### 3.9 场景快照流程

1. 保存快照：写入 `liteScheduler.scenario.snapshots.v1`。
2. 加载快照：替换当前场景并刷新看板。
3. 快照支持重命名、删除。

### 3.10 导出流程

1. 将当前已排产明细导出为 `.xls`（HTML table 方式）。
2. 按模式显示单位：
   1. 按量模式：数量。
   2. 按天模式：天数。

## 4. 核心业务规则（当前实现）

1. 订单排序以 `orderSeq` / `orderNo` 为主。
2. 按量模式优先级：锁定片段 > 指定产线工作量 > 自动补齐。
3. 按天模式每条产线每日容量固定视为 1 天槽位，基础产能数值不参与计算。
4. “推进一天”会滚动窗口并记录模拟日志。
5. 当前订单优先级字段在本模块内固定为 `NORMAL`（未形成加急差异策略）。

## 5. 与主系统关系

1. 该模块当前未打通主后端 `/api`、`/v1`、`/internal/v1/internal` 排产接口。
2. 当前更适合作为“本地沙盘/快速方案工具”。
3. 若后续纳入主系统，需要补齐：
   1. 场景持久化与多人协作模型。
   2. 与主排产版本的映射关系。
   3. 规则口径对齐与统一审计链路。
