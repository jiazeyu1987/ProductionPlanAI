# 璞慧生产数据流程梳理（独立文档）

更新时间：2026-03-28  
范围：`LiteSchedulerPage` 本地场景数据与 `liteSchedulerEngine` 计算数据流

## 1. 数据边界说明

璞慧生产当前为前端本地模块，数据边界如下：

1. 事实源（Source of Truth）：浏览器内存场景对象 + `localStorage`。
2. 不依赖后端数据库、后端 API、外部系统。
3. 所有排产结果为“即时计算结果”，非后端持久化版本。

## 3. 本地存储模型

1. 当前场景：`liteScheduler.scenario.v1`
2. 场景快照：`liteScheduler.scenario.snapshots.v1`

读写流程：

1. 页面加载：`localStorage -> JSON.parse -> normalizeLiteScenario -> state`
2. 页面编辑：`state 变更 -> normalizeLiteScenario -> localStorage`
3. 快照操作：独立键读写，不影响主键结构

## 4. 计算数据流

## 4.1 通用管线

1. 原始场景输入（UI 操作/快照加载）
2. `normalizeLiteScenario`（数据归一化与纠偏）
3. `buildLiteSchedule`（按模式计算）
4. UI 消费 `lineRows/orderRows/summary` 渲染表格与日历

## 4.2 按量模式（`QTY_CAPACITY`）数据流

1. 构建 `dates` + 各线每日容量（基础值 + 覆盖值）。
2. 建立订单剩余量映射 `orderRemaining`。
3. 先消费锁定片段（`source=LOCKED`）。
4. 再执行自动分配：
   1. 优先满足订单指定产线工作量（`source=AUTO_LINE`）。
   2. 剩余容量做动态补齐（`source=AUTO`）。
5. 汇总产线利用率、订单剩余、延期与瓶颈线。

## 4.3 按天模式（`DURATION_MANUAL_FINISH`）数据流

1. 每条产线每日容量固定按 `1` 处理。
2. 订单按 `linePlanDays` 逐线排段。
3. 若设置 `manualFinishByLineOrder`，以手动结束日期覆盖该段结束点。
4. 若未报结束且已过当前锚点，会做顺延扩展（`finishSource=EXTENDED`）。
5. 输出订单“完成状态/实际完成日期/延期天数”等字段。

## 5. 推进一天的数据变化

调用 `advanceLiteScenarioOneDay` 后，执行以下变更：

1. `horizonStart` -> 下一工作日
2. 订单 `completedDays` 增量更新
3. 锁定片段按当日消耗扣减、窗口前移
4. `simulationLogs` 头插当日完成量（保留最近 30 条）
5. 返回 `daySummary`（完成量、剩余量、延期数、提示语）

## 6. UI 层到数据层映射（关键动作）

1. 新增/编辑订单：更新 `orders` + `nextOrderSeq`
2. 新增/删除产线：更新 `lines`（含容量覆盖）
3. 新增/删除锁定：更新 `locks`
4. 日历休息/排产切换：更新 `dateWorkModeByDate`
5. 手动报结束：更新 `manualFinishByLineOrder`
6. 推进一天：批量变更 `horizonStart/orders/locks/simulationLogs`
7. 重置场景：回到 `createDefaultLiteScenario`

## 7. 当前数据实现约束

1. 单浏览器单用户本地模型，不支持协同编辑。
2. 刷新/换设备依赖 `localStorage`，无服务端备份。
3. 当前 `priority` 在该模块中固定为 `NORMAL`，未形成差异化调度权重。
4. 数据与主排产版本无统一主键与映射关系。

## 8. 后续重构建议（璞慧生产独立方向）

1. 抽象 `scenario` 版本号与迁移器（schema migration）。
2. 引入可选后端保存接口（本地/云端双存储策略）。
3. 建立与主系统排产版本的映射字段（如 `externalVersionNo`）。
4. 把算法输入输出抽成独立 DTO，便于与后端服务对接与回放测试。
