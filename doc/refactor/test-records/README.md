# 测试记录目录说明

本目录用于保存“每次小步重构后的测试验证记录”。

## 1. 文件命名建议

1. `YYYYMMDD-模块-任务编号.md`  
示例：`20260328-schedule-BE-A-01.md`

## 2. 记录模板

1. 重构步编号：
2. 波次（Wave）：
3. 变更模块：
4. 变更文件：
5. 变更说明：
6. 契约影响：`无 / 有（说明）`
7. 执行测试命令：
8. 测试结果：
9. 风险与回滚：
10. 是否允许进入下一步：`YES/NO`

## 3. 使用要求

1. 每个任务编号至少对应一条记录。
2. 测试失败的记录也要保留，便于复盘。
3. 未补测试记录不得视为该步已完成。
