# PRD（排产系统）

## 1. 产品定位
- 产品名称：排产协同系统  
- 产品目标：以订单交付为目标，实现“计划-执行-反馈-重排”闭环。  
- 核心边界：P0 不做库存精算，库存决策留在 ERP/MRP。  
- 排产对象：以 ERP 生产订单为排产对象（不是生产用料清单）。  
- 产能口径：按班次计算，`1班次=12小时`，每日开班数为`1`或`2`。  

## 2. 目标用户
- 计划员：编排周计划、调序、发布版本。  
- 生产管理：查看工序负荷与延期风险。  
- 采购/外围协同人员：接收计划并反馈执行状态。  
- 管理层：看交付达成率、异常、瓶颈。  

## 3. 用户故事（关键）
- 作为计划员，我希望自动生成初版排程，再人工微调，避免纯手工编排。  
- 作为计划员，我希望插单后系统自动评估影响范围。  
- 作为生产管理，我希望报工后系统自动刷新预计完工时间。  
- 作为管理层，我希望看到延期风险和当前计划版本差异。  

## 4. 功能需求（P0）

### 4.1 订单待排池
- 支持按状态、交期、客户、产品、加急筛选。
- 支持冻结/解冻、锁单标记。
- 支持查看订单来源链路（销售订单 -> 计划订单 -> 生产订单）。
- 支持查看评审状态与投放状态（计划订单投放后才进入生产订单待排池）。

### 4.2 排产规则引擎
- 优先级因子：交期、加急、下单时间、客户等级、人工权重。
- 规则支持按产品线配置。
- 支持手动覆盖系统排序。

### 4.3 有限产能排产
- 排产输入主体为生产订单；生产用料清单仅用于追溯与物料协同展示。
- 依据 MES 工艺路线与设备产能。
- 支持串行/并行工序关系。
- 支持人力技能、设备能力、物料可用三类约束联合校验。
- 输出订单级和工序级预计开始/完成时间。

### 4.4 调度台
- 列表+甘特双视图。
- 拖拽调序、批量顺延、插单。
- 发布前影响预览（受影响订单数、预计延期数）。

### 4.5 版本管理
- 每次发布产生版本号（V1/V2...）。
- 支持版本差异对比（顺序、时间、产能占用变化）。
- 支持回滚到历史版本。

### 4.6 执行反馈与重排
- 接收 MES 报工后自动更新进度。
- 触发重排条件可配置（延期阈值、欠产阈值、异常事件）。
- 生成重排候选版本（不直接覆盖已发布版本）。
- 默认仅重排未完成工序；冻结/锁单任务不参与自动重排。

### 4.7 预警与看板
- 预警类型：延期风险、超负荷、报工异常、关键工序堵塞。
- 告警通道：钉钉、邮件、系统内消息。
- 看板指标：计划达成率、准交率、延期订单数、重排次数。

### 4.8 审计留痕
- 记录所有调整操作（操作者、时间、字段、前后值、原因）。
- 支持按订单追溯完整调整链路。
- 调序、发布、回滚、解冻权限可配置，并对关键操作强制审计。

### 4.9 同步机制
- 支持两类同步：触发同步（事件触发）+ 每晚定时补偿同步。
- 触发同步用于准实时更新，夜间补偿用于漏数修复与对账。

## 5. 非功能需求
- 可用性：核心操作可稳定完成（发布、重排、查询）。
- 性能：单次排产计算 P95 <= 5 分钟。
- 安全：操作权限分级、全链路日志。
- 可追溯：任意订单可追溯到版本与操作记录。

## 6. 业务规则（初版）
1. 未评审通过或已冻结订单不得参与自动排产。  
2. 锁单默认不参与自动重排。  
3. 同一工序同一时段不得超设备可用能力。  
4. 报工后仅重算“未完成工序”。  
5. 计划订单未投放时，不得进入生产订单排产池。  
6. 生产订单与生产用料清单一一对应，但排产不直接基于生产用料清单。  
7. 可开工前需同时满足人力、设备、物料约束。  
8. 日可排产能公式：`日可排产能 = capacity_per_shift * 当日开班数（1或2）`。  

## 7. 验收标准（P0）
1. 系统可完成周计划编排、发布与版本对比。  
2. 支持插单并可评估影响。  
3. 报工后自动更新进度并触发重排候选版本。  
4. 所有关键改动可留痕追溯。
