# 测试方案文档（排产系统）

## 1. 测试目标
- 验证系统可稳定完成“接单 -> 排产 -> 发布 -> 报工反馈 -> 重排”的闭环。
- 验证关键规则正确性与可追溯性。

## 2. 测试范围
- 在范围内：订单池、规则引擎、排产引擎、调度台、版本管理、报工反馈、预警、回写。
- 不在范围内：库存精算优化、复杂成本优化。

## 3. 测试类型
1. 单元测试：规则计算、状态机、时间窗口计算。  
2. 集成测试：ERP/MES接口入站与回写、幂等去重。  
3. 端到端测试：完整业务流程（含班次与重排）。  
4. 回归测试：规则变更后的主流程验证。  
5. UAT：业务场景验收。  

## 4. 关键测试场景

### 4.1 订单准入
- 已审核且评审通过订单可进入待排池。
- 冻结订单不可自动排产。
- 计划订单未投放时不得进入生产订单待排池。
- 计划订单投放后应能正确生成并接入生产订单。

### 4.2 排产计算
- 排产仅基于生产订单，不直接基于生产用料清单。
- 同一工序不可超产能占用。
- 串行工序先后关系正确。
- 并行工序可并发安排。
- 单次排产计算 P95 <= 5 分钟。
- 班次口径正确：`1班次=12小时`，`1天1班/2班`产能计算一致。
- 人机料约束正确：任一约束不足时任务不可开工且有原因码。
- 同一设备同一时段只允许一个工艺任务。
- 同一员工同一时段不允许重复分配。

### 4.3 人工调度
- 拖拽调序后结果可保存。
- 锁单后自动重排不影响锁定订单。
- 插单后影响范围计算正确。

### 4.4 版本发布
- 发布后版本状态正确切换。
- 版本差异可查询。
- 回写ERP结果完整。

### 4.5 报工与重排
- 报工到达后完成率自动更新。
- 延期阈值触发重排候选版本。
- 异常事件可触发预警。
- 轻微偏差触发局部重排，重大异常触发全局重排。
- 冻结窗内任务不被自动重排改动。
- 工序完工后可生成唯一WIP批次并可追踪。

### 4.6 同步与补偿
- 触发同步链路可在事件到达后完成入站与处理。
- 夜间定时补偿可修复漏数并输出差异结果。
- 幂等去重生效（重复消息不重复入账）。

## 5. 数据质量测试
- 字段完整性、主键唯一性、日期数值合法性。
- 跨表关联一致性（订单映射、来源追溯）。

## 6. 缺陷管理
- 缺陷等级：阻断/高/中/低。
- 闭环要求：复现步骤、影响范围、修复验证记录。

## 7. UAT通过标准
1. 关键流程无阻断缺陷。  
2. 计划发布与回写可稳定完成。  
3. 报工反馈与重排逻辑符合业务预期。  
4. 关键报表与预警可正常使用。  
5. 单次排产计算性能达标（P95 <= 5 分钟）。  
6. 班次口径与人机料约束相关用例全部通过。
