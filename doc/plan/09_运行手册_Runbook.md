# 运行手册（Runbook）

## 0. 当前系统口径（2026-03-23）
1. 一键启动：仓库根目录执行 `.\run_stack.ps1`（或 `.\run_stack.bat`）。
2. 默认地址：前端 `http://localhost:5932`，后端 `http://localhost:5931`。
3. 健康检查：`GET http://localhost:5931/api/health`。
4. 运行日志目录：`code/logs/`（`backend-local.*.log`、`frontend-dev.*.log`）。
5. 后端默认 profile：`local`（H2 内存库）；PostgreSQL 联调模式按需启用。

## 1. 日常操作清单
1. 检查 ERP/MES 触发同步是否正常。  
2. 检查夜间补偿同步任务结果与差异报告。  
3. 检查待排池是否有异常积压。  
4. 检查当天预警（延期/超负荷/报工异常）。  
5. 检查发布版本是否回写成功。  
6. 检查报工入站延迟与重排触发情况。  
7. 检查当日开班数（1/2班）与排产产能是否一致。  
8. 检查关键工艺人力覆盖、设备可用与缺料占比。  

## 2. 关键监控指标
- 数据同步延迟（ERP、MES）
- 触发同步成功率与补偿同步修复率
- 待排订单数量与增长趋势
- 排产计算耗时
- 版本发布成功率
- ERP回写成功率
- 预警数量与处理时长
- 班次开班率与班次产能利用率
- 关键工艺人力覆盖率、设备可用率、缺料任务占比

## 3. 常见告警与处理

### 3.1 同步中断
- 现象：长时间无新增数据。
- 处理：检查触发事件源、接口连通、认证、增量游标、失败队列；必要时执行夜间补偿任务提前重跑。

### 3.2 发布失败
- 现象：计划发布后未回写ERP。
- 处理：查看回写错误码，重试幂等回写，必要时人工补偿。

### 3.3 报工异常
- 现象：报工数量异常或工序缺失。
- 处理：核对MES原始记录、工艺主数据、订单关联关系。

### 3.4 重排异常
- 现象：重排候选版本与业务预期偏差较大。
- 处理：核对规则配置、锁单状态、工序串并行配置。

### 3.5 班次产能异常
- 现象：计划产能与现场班次不一致。
- 处理：核对班次日历、开班标记、`capacity_per_shift` 与当日开班数。

### 3.6 资源不可执行
- 现象：任务被系统判定不可开工。
- 处理：核对技能矩阵、设备工艺能力矩阵、物料齐套状态与更新时间。

## 4. 运行规则
- 未入系统的口头指令不生效。  
- 锁单变更必须写明原因并留痕。  
- 临时规则使用后需评估是否固化为配置。  
- 预警默认同时支持钉钉、邮件、系统内消息三通道。  

## 5. 变更管理
- 规则变更前需执行回归测试。  
- 配置变更必须版本化。  
- 发布后必须观察关键指标稳定性。  

## 6. 备份与恢复规范
- 每日备份业务库与关键配置。  
- 预留最近可用快照用于快速恢复。  
- 定期演练恢复流程，确保可执行。
