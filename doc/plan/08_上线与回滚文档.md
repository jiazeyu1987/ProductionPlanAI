# 上线与回滚文档（排产系统）

## 0. 当前系统口径（2026-03-23）
1. 一键启动：仓库根目录执行 `.\run_stack.ps1`（或 `.\run_stack.bat`）。
2. 默认地址：前端 `http://localhost:5932`，后端 `http://localhost:5931`。
3. 健康检查：`GET http://localhost:5931/api/health`。
4. 运行日志目录：`code/logs/`（`backend-local.*.log`、`frontend-dev.*.log`）。

## 1. 上线前检查
1. 接口联调通过（ERP/MES入站、ERP回写）。  
2. 数据口径确认通过（关键字段完整）。  
3. UAT通过且关键缺陷已关闭。  
4. 发布包与配置项已冻结。  
5. 回滚脚本与备份已准备。  
6. 脚本启动栈健康检查通过（frontend `5932` / backend `5931`）。  
7. 班次日历初始化完成（`1班次=12小时`，日班/夜班可配置）。  
8. 设备-工艺能力矩阵、员工技能矩阵已完成首轮导入。  
9. 物料可用状态链路可用（至少支持齐套/缺料标识）。  

## 2. 上线步骤
1. 备份生产数据库与关键配置。  
2. 部署应用服务（api内含排产计算与任务调度）。  
3. 执行数据库变更脚本。  
4. 启动服务并通过健康检查（`/api/health`）。  
5. 执行一次全量初始化同步。  
6. 执行增量同步验证。  
7. 验证触发同步与夜间补偿任务均可执行（MVP由api内置任务执行器承载）。  
8. 试发布一个小范围计划版本。  
9. 验证回写与预警链路（钉钉/邮件/站内）。  
10. 开启正式运行。  
11. 完成首日班次演练（1班）与扩容演练（2班）。  

## 3. 上线后核验
- 核验订单池数量与ERP一致。  
- 核验工艺与设备数据可用。  
- 核验排产发布与ERP回写成功。  
- 核验报工进入后进度变化正确。  
- 核验班次产能口径与排产结果一致。  
- 核验关键工艺的人机料约束命中逻辑正确。  

## 4. 回滚触发条件
- 核心流程不可用（无法发布计划/无法回写）。  
- 数据错乱（主键冲突、关联大面积失效）。  
- 严重性能问题影响业务连续操作。  
- 同步链路中断超过30分钟或连续3次排产计算超时。  

## 5. 回滚步骤
1. 暂停新发布动作与增量入站。  
2. 切换只读，保护现状数据。  
3. 回退应用版本。  
4. 回退数据库结构与数据快照。  
5. 恢复旧流程（Excel+原有操作）。  
6. 记录问题并触发故障复盘。  

## 6. 应急沟通要点
- 对外统一说明影响范围与临时方案。  
- 明确恢复完成判定标准。  
- 保留完整操作日志用于复盘。
