# 接口规格文档（排产系统，现行）
> 生效日期：2026-03-22  
> 本文与 `openapi/scheduling-v1.yaml`、`doc/develop/04_接口规格文档.md`、`doc/develop/11_接口契约与Schema规范_P0.md` 保持一致。

## 1. 接口总览
- 路径约定：
  - 本文中的接口路径为 OpenAPI 相对路径。
  - 实际联调调用时必须补全契约前缀：外部接口使用 `/v1` + 相对路径（例如 `GET /v1/erp/sales-order-lines`）。
  - 本文“内部”回写事件同样属于外部契约路径，完整调用示例：`POST /v1/internal/replan-jobs`。
- 入站（读取）：
  - ERP：订单主业务事实、ERP原始头表/行表、准入控制、MRP关联、交付进度
  - 报表：周计划、月计划、周/月计划模板导出
  - MES：设备、工艺、报工、设备工艺能力、员工技能、班次日历
- 出站（回写）：
  - ERP：排产结果、排产状态
  - 内部：在制品事件、重排任务

## 2. 数据交换方式
- 默认方式：REST API（JSON）。
- 兼容方式：批量文件（CSV/XLSX）。
- 同步模式：首次全量 + 增量同步（`updated_after`）+ 夜间补偿同步。

## 4. 出站接口定义（现行）

### 4.1 回写ERP-排产结果
- 接口：`POST /erp/schedule-results`
- 请求体核心字段：`request_id` `schedule_version` `items[].order_no` `items[].order_type` `items[].plan_start_time` `items[].plan_finish_time` `items[].priority` `items[].lock_flag`

### 4.2 回写ERP-排产状态
- 接口：`POST /erp/schedule-status`
- 请求体核心字段：`request_id` `schedule_version` `items[].order_no` `items[].status`

## 5. 通用约定
- 时间格式：ISO 8601（含时区），时区统一 `Asia/Shanghai`。
- 分页参数：`page`（从1开始）、`page_size`（默认200，最大1000）。
- 增量参数：`updated_after`（按 `last_update_time asc`）。
- 写接口幂等键：`request_id + 业务主键`。
- 返回头：必须包含 `x-request-id`。

## 6. 错误码与重试
- `400` 参数错误（不重试）
- `401/403` 鉴权失败（人工处理）
- `404` 资源不存在（人工核对）
- `409` 并发冲突（短重试）
- `429` 限流（指数退避重试）
- `5xx` 服务异常（自动重试）

## 7. 验收标准
1. ERP主业务接口（销售/生产）不展开动态海量列，字段稳定可用。
2. ERP原始头表/行表必须可回放，`erp_header_json` 与 `erp_line_json` 可完整追溯。
3. 主业务表与原始表可通过 `erp_record_id`、`erp_line_id` 关联。
4. 周计划/月计划接口字段必须与金标准模板字段一致，导出格式与文件后缀一致（周 `xlsx`、月 `xls`）。
5. 前后端联调通过，接口字段、枚举、路径与 OpenAPI 一致。
