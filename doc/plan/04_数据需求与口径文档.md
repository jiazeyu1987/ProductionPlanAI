# 数据需求与口径文档（排产系统）

## 1. 数据来源矩阵

### 1.1 ERP（K3）
- 销售订单/订单行：订单号、行号、客户、物料、数量、交期、发货日期、状态。
- 计划订单：计划订单号、投放类型（采购/委外/生产）、投放状态、投放时间。
- 生产订单：生产单号、来源销售单、来源计划订单、计划数量、计划开完工、状态。
- 生产用料清单：清单号、对应生产单号、物料编码、需求数量（用于追溯，不作为排产输入主体）。
- 采购执行链路：采购申请、采购订单、收料通知单、采购入库单、即时库存结果。
- 控制状态：审核状态、冻结状态、关闭状态、可排产标识。
- MRP关联：采购申请号、委外申请号、投放时间。
- 交付状态：入库数量、发货数量、检验状态、交付状态。

### 1.2 MES
- 设备主数据：设备编码、可用状态、所属产线、产能参数。
- 工艺主数据：产品-工序路线、串并行关系、标准节拍。
- 报工数据：订单、工序、报工数量、报工时间、班组/人员、异常码。
- 设备-工艺能力矩阵：设备编码、工序编码、可用标记、产能系数。
- 员工技能矩阵：员工、工序、技能等级、效率系数、生效状态。
- 班次日历：日期、班次（日/夜）、起止时间、开班标记。

### 1.3 人工录入
- 插单申请、优先级调整、锁单指令、异常处理决策。
- 口头信息必须转为系统记录，不允许长期口头生效。

## 2. 主键与关联键
- 销售订单行主键：`sales_order_no + line_no`
- 计划订单主键：`plan_order_no`
- 生产订单主键：`production_order_no`
- 生产用料清单主键：`material_list_no + material_code`
- 订单跨表关联键：`order_no + order_type`
- MRP链路关联键：`source_sales_order_no + source_line_no -> plan_order_no -> production_order_no`
- 工序任务主键：`schedule_version + order_no + process_code`
- 设备工艺能力主键：`equipment_code + process_code`
- 员工技能主键：`employee_id + process_code`
- 班次日历主键：`calendar_date + shift_code (+ workshop_code)`

## 3. 状态口径定义
- `待排`：已通过准入但未进入发布版本。  
- `已排`：已在草稿版本中排定。  
- `已发布`：版本正式发布并生效。  
- `执行中`：已有报工且未完成。  
- `完工`：订单计划量已全部完成并关闭。  
- `异常`：触发预警且需要人工处理。  
- `冻结`：禁止自动排产与自动重排。  

## 4. 交期口径定义
- `期望交期`：业务/市场期望完成日期。  
- `承诺交期`：评审后对内外承诺日期。  
- `要求发货日期`：交付动作的目标日期。  
- 排产输出默认基于承诺交期，发货日期用于交付协同。  

## 4.1 产能口径定义（已确认）
- 班次口径：`1班次=12小时`。
- 开班规则：常规`1天1班`，产能紧张时可`1天2班（日班+夜班）`。
- 日可排产能：`日可排产能 = capacity_per_shift * 当日开班数（1或2）`。
- 可开工前提：人力、设备、物料三类约束同时满足。

## 5. 数据质量规则
1. 必填字段不得为空（订单号、数量、交期、状态、更新时间）。  
2. 主键不得重复。  
3. 日期字段必须可解析。  
4. 数量字段必须为数值且不可为负。  
5. 0/1标识字段必须合法（如加急、冻结、可排产）。  
6. 生产订单来源销售订单必须可追溯。  
7. 生产订单与生产用料清单必须一一对应（缺失需告警）。  
8. 即时库存仅作为事实数据同步，不参与P0排产求解。  
9. 班次日历缺失或班次时段非法时，禁止发布计划版本。  
10. 技能矩阵无覆盖、设备工艺能力无覆盖、缺料状态未确认时，任务不得进入可执行清单。  

## 6. 增量同步口径
- 以 `last_update_time` 为增量游标。
- 首次全量，后续采用“触发同步 + 夜间定时补偿同步”。
- 对账维度：订单数、状态分布、关键数量总和。

## 7. 手工数据治理规则
- 每条人工指令都必须包含：发起人、生效时间、原因、影响范围。
- 人工指令默认仅影响当前发布周期，长期规则需转配置。
- 口头通知未入系统视为无效指令。

## 8. 数据库补齐清单（P0必补，含作用）
1. `product`：产品主数据维度，统一产品编码与规格口径。  
2. `process`：工艺字典，统一工艺编码与关键工序标识。  
3. `organization_unit`：工厂/车间/工段/产线/班组层级关系。  
4. `employee`：员工主档，与技能矩阵解耦。  
5. `product_route_header`：产品工艺路线版本头与生效区间。  
6. `route_step_dependency`：工序依赖边（串并行与前后置约束）。  
7. `process_resource_requirement`：工艺标准资源需求（人/机/标准班产）。  
8. `schedule_task_resource`：任务级设备/人员占用明细（冲突校验）。  
9. `replan_job`：重排任务主表（触发原因、范围、结果版本）。  
10. `wip_lot_event`：中间在制批次流转事件（创建/转序/HOLD/报废/消耗）。  
11. `integration_inbox`：ERP/MES 入站原始消息与处理状态。  
12. `integration_outbox`：ERP 回写与通知出站可靠投递。  
13. `idempotency_ledger`：幂等账本（防重复入账与重复回写）。  
14. `sync_checkpoint`：增量同步游标（断点续传）。  
15. `dispatch_command_approval`：高风险调度指令审批链。  
16. `schedule_publish_log`：版本发布/回滚日志。  
17. `equipment_downtime_event`：设备停机事件事实（用于“关键设备停机 > 30 分钟”重排触发）。  
18. `employee_shift_availability`：员工班次在岗事实（避免“会技能但不在岗”导致排产失真）。  
19. `schedule_task_change_log`（增强项）：任务级变更明细日志。  
20. `alert_event`：预警事件主表（延期风险、超负荷、报工异常等）。  
21. `rule_param_version`：规则参数版本表（权重、阈值、冻结窗、班次策略）。  
22. `notification_delivery_log`（增强项）：钉钉/邮件/系统消息投递日志。  
23. `sales_order_line_fact`：销售订单行原始事实表（来源追溯与客户维度分析）。  
24. `production_order_fact`：生产订单原始事实表（与待排池解耦）。  
25. `schedule_control_fact`：准入控制事实表（评审/冻结/可排产/关闭）。  
26. `mrp_result_link`：MRP 结果链路事实表（采购/委外/生产关联）。  
27. `production_material_list_item`：生产用料清单明细表（齐套核查与追溯）。  
28. `delivery_progress`：交付进度事实表（入库/发货与准交统计）。  
29. `order_priority_factor`：订单评分快照表（落地客户优先级与人工权重）。  

说明：详细字段定义与索引见 `doc/develop/05_数据库设计文档.md`。

## 9. 关键关系链（评审口径）
1. 订单链：`sales_order_line_fact -> production_order_fact -> order_pool -> schedule_task -> reporting_fact -> wip_lot`。  
2. 准入链：`schedule_control_fact -> order_pool`（决定是否进入可排清单）。  
3. 工艺链：`product -> product_route_header -> process_route(route_no) -> route_step_dependency -> process`。  
4. 资源链：`process -> process_resource_requirement`，并与 `equipment_process_capability`、`employee_skill` 联合校验。  
5. 版本链：`rule_param_version -> schedule_version -> schedule_task`，发布与回滚落 `schedule_publish_log`。  
6. 重排链：`equipment_downtime_event / reporting_fact / 插单事件 -> replan_job -> schedule_version`。  
7. 预警链：`schedule_task / reporting_fact -> alert_event -> notification_delivery_log`。  
8. 同步链：`integration_inbox -> 业务事实表`，`业务结果 -> integration_outbox`，幂等由 `idempotency_ledger` 保证。
