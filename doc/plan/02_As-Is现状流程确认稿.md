# As-Is现状流程确认稿（排产）
> 文档属性：历史现状记录（As-Is），不作为当前实施口径。
> 当前实施口径以 `doc/00_现行口径基线_2026-03-21.md` 为准。

## 1. 当前流程步骤（1-16）
1. 市场/外贸在 K3 新建销售订单或预测单，并完成审核。  
2. 市场发起钉钉交期评审。  
3. 计划/采购/生产参与评审，确认可承诺交期。  
4. 评审通过后，市场邮件通知“订单可执行”。  
5. 计划在 K3 做 MRP 运算，生成计划订单列表（采购申请/委外/生产）。  
6. 计划订单执行“投放”（等同提交审核）后，触发生产订单与采购申请。  
7. 生产订单生成时同步创建生产用料清单，二者一一对应。  
8. 计划员将生产订单汇总到周计划 Excel 与生产计划总表。  
9. 按交期、加急、下单先后、客户优先级进行人工排序。  
10. 排产对象是生产订单；排产时参考 MES 工艺与设备信息，但大量依赖经验粗排。  
11. 形成周计划后通过表格/邮件/沟通下发到车间、外围、采购。  
12. 采购申请下推采购订单，采购订单按计划执行。  
13. 采购订单入库后形成即时库存，并联动收料通知单与采购入库单。  
14. 车间在 MES 报工，车间与外围通过交接单流转。  
15. 计划员根据报工和异常手工调整 Excel（顺延/插单/改优先级）。  
16. 完工后在 K3 走检验、入库、调拨、发货，并以本周实绩滚动下一周排产。  

## 2. 系统触点与数据来源
- ERP(K3)：销售订单、计划订单、生产订单、采购申请/采购订单、交期、审核/冻结、MRP关联、交付状态。  
- MES：设备、工艺、工序、报工。  
- Excel：周计划与临时调整载体。  
- 人工沟通：加急插单、临时改序、异常决策。  

## 3. 现状痛点
- 排产版本多且分散，难追溯“最终有效版本”。  
- 口头改动未结构化，容易造成执行偏差。  
- 报工到计划的反馈链路有延迟，重排效率低。  
- 工序级能力与排产规则未完全系统化。  

## 4. 历史需确认事项（已失效，仅供追溯）
1. 可排产口径是否为：`已审核 + 评审通过 + 未冻结 + 未关闭`。  
2. 当前是否允许未评审订单进入周计划。  
3. 插单是否必须审批，还是由计划员直接调整。  
4. 锁单后是否允许系统自动重排。  
5. 哪些异常会触发当天重排（设备停机/质量异常/人力缺口）。  
6. 外围包装与车间排产是否同一计划版本管理。  

## 5. 本确认稿输出用途
- 作为 To-Be 设计基线；后续所有需求与设计变更都需对照本稿。
